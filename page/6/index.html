hexo添加分类和标签

转载自hexo分类与tags配置
pyartist的回答

添加标签和分类格式 

tags:
  - tag1
  - tag2
categories: xxx

冒号后面要有空格
主题的配置文件和站点的配置文件tags和categories的注释要打开

添加标签

hexo new page tags
确认站点配置文件里有tag_dir: tags
确认主题配置文件里有tags: /tags
编辑站点的source/tags/index.md，添加 

title: tags
date: 2015-10-20 06:49:50
type: "tags"
comments: false

添加分类

hexo new page categories
确认站点配置文件里有category_dir: categories
确认主题配置文件里有categories: /categories
编辑站点的source/categories/index.md，添加 

title: categories
date: 2015-10-20 06:49:50
type: "categories"
comments: false

搭建博客

搭建 hexo+github 的流程&问题

搭建blog参考了：
教你免费搭建个人博客，Hexo&Github
Hexo常见问题解决方案
有关使用 Hexo 和 GitHub 搭建博客，出现 hexo -d 报错如何解决？

一、配置GitHub

首先登录GitHub注册账号
然后create new repository
因为我的名字是t51645所以repository name 填写t51645.github.io

#二、环境安装#
安装Node.js
安装Git
安装完毕后打开Git，可以从桌面右击打开

然后配置git用户信息 

git config --global user.name "用户名"  
git config --global user.email "邮箱"


然后再安装Hexo 

npm install -g hexo-cli

三、设置

在自己的D盘（不限）目录下新建目录Blog，再在Blog目录下右击选择 Git Bash Here 打开Git命令行
再输入初始化blog 

hexo init blog

成功提示如下： 

INFO  Start blogging with Hexo!

再执行如下命令生成博客启动本地服务器 

hexo generate  
hexo server

提示 

INFO  Start processing  
INFO  Hexo is running at http://localhost:4000/. Press Ctrl+C to stop.

然后在localhost:4000中就能看到自己的博客了

这里4000端口可能被占用了，可以用如下命令换端口 

hexo s -p 5000

这样就可以用5000端口了

友情提醒：
如果出现hexo命令失效可以参考hexo常见问题解决方案中的hexo命令失效解决方案

接下来再输入 

ssh-keygen -t rsa -C "Github的注册邮箱地址"

得到 

Your public key has been saved in C:/Users/Administrator/.ssh/id_rsa.pub

再找到id_rsa.pub,复制里面的内容然后进入GitHub new SSH key
New SSH key ——Title：blog —— Key：输入刚才复制的—— Add SSH key

四、配置博客

在blog目录下，用支持UTF-8的程序打开_config.yml文件如sublime不然输入的中文会成乱码，修改参数信息 

# Hexo Configuration
## Docs: http://hexo.io/docs/configuration.html
## Source: https://github.com/hexojs/hexo/
# Site #站点信息
title:  #标题
subtitle:  #副标题
description:  #站点描述，给搜索引擎看的
author:  #作者
email:  #电子邮箱
language: zh-CN #语言
# URL #链接格式
url:  #网址
root: / #根目录
permalink: :year/:month/:day/:title/ #文章的链接格式
tag_dir: tags #标签目录
archive_dir: archives #存档目录
category_dir: categories #分类目录
code_dir: downloads/code
permalink_defaults:
# Directory #目录
source_dir: source #源文件目录
public_dir: public #生成的网页文件目录
# Writing #写作
new_post_name: :title.md #新文章标题
default_layout: post #默认的模板，包括 post、page、photo、draft（文章、页面、照片、草稿）
titlecase: false #标题转换成大写
external_link: true #在新选项卡中打开连接
filename_case: 0
render_drafts: false
post_asset_folder: false
relative_link: false
highlight: #语法高亮
  enable: true #是否启用
  line_number: true #显示行号
  tab_replace:
# Category & Tag #分类和标签
default_category: uncategorized #默认分类
category_map:
tag_map:
# Archives
2: 开启分页
1: 禁用分页
0: 全部禁用
archive: 2
category: 2
tag: 2
# Server #本地服务器
port: 4000 #端口号
server_ip: localhost #IP 地址
logger: false
logger_format: dev
# Date / Time format #日期时间格式
date_format: YYYY-MM-DD #参考http://momentjs.com/docs/#/displaying/format/
time_format: H:mm:ss
# Pagination #分页
per_page: 10 #每页文章数，设置成 0 禁用分页
pagination_dir: page
# Disqus #Disqus评论，替换为多说
disqus_shortname:
# Extensions #拓展插件
theme: landscape-plus #主题
exclude_generator:
plugins: #插件，例如生成 RSS 和站点地图的
- hexo-generator-feed
- hexo-generator-sitemap
# Deployment #部署，将 lmintlcx 改成用户名
deploy:
  type: git
  repo: 刚刚github创库地址.git
  branch: master

注意每个参数的:后都有空格不然会出错

配置 

deploy: 
  type: git
  repo: git@github.com:t51645/t51645.github.io.git
  branch: master

将repo按格式改成自己的即可

五、发表文章

在Git Bash中输入 

hexo new "test"

提示创建文章在 

INFO  Created: D:\Blog\source\_posts\test.md

可以在test中修改自己想要内内容，这里文章使用的是Markdown语法
接下来可以输入 

hexo clean
hexo generate
hexo server

观察是否发布了新的文章
最后一步，发布到网上： 

hexo deploy

接下来就可以在https://t51645.github.io/查看自己的博客是否发布
如果出现错误可以参考https://www.zhihu.com/question/38219432中邹世杰的回答，我是这样解决的
